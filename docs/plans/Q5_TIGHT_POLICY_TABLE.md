## Q5 꽉낌 분기 정책표 (고정안)

목적:
- Q5-3 응답을 사이즈 결정의 직접 기준으로 단순화
- 과보정 위험이 있는 `꽉낌 추정 보정 +1` 의존도를 제거
- 대화 길이를 줄이고 추천 일관성을 높임

적용 범위:
- 상태: `Q_FIT_EXP=3(볼이 꽉 껴서 불편함)` 이후 분기
- 관련 질문: `Q5-2`, `Q5-3`

## 1) Q5-3 의사결정 테이블

| Q5-3 응답 | 해석 | 사이즈 추천 원칙 | 가공/보정 원칙 | 안내 문구 핵심 |
|---|---|---|---|---|
| 1. 네 (한 치수 업 시 헐떡임) | 업사이즈 리스크 큼 | 보수 추천(작은/기준 쪽) | 적용점수 기반 보정, 앞코 너비 보정 필요 명시 | “업 시 헐떡임이 있어 보수 사이즈 권장” |
| 2. 아니요 (업해도 헐떡임 없음) | 업사이즈 수용 가능 | 같은 핏라인에서 한 사이즈 업 권장 (예: 225→230) | 과보정 점수 가중 없이 필요한 보정만 명시 | “업사이즈 수용 가능해 한 치수 업 권장” |
| 3. 신어본 적 없음 (기존: 모름·해본 적 없음) | 정보 불확실 | 무지외반 통증이 심하면 한 사이즈 업 우선 검토 | 작은 사이즈 선택 시 발볼늘림 3단계 + 앞코 너비 보정 안내 | “한 치수 더 크게는 안 신어봤어요” 등 → 안전측 분기 안내 |

## 2) 점수 정책(수정안)

- 기존 `꽉낌 추정 보정 +1`은 기본 정책에서 제거
- 점수는 기존 증상/핏 입력으로 계산하되, Q5-3은 **직접 분기 트리거**로 사용
- 즉, Q5-3은 “점수 가중”보다 “사이즈 방향 결정”에 우선 적용

## 3) 예시 가이드 (관리자 확인용)

- Q5-3 = `2(아니요)`  
  - 추천사이즈: 한 사이즈 업 (예: `230mm`)  
  - 설명: 업사이즈 시 헐떡임이 없으므로 같은 핏라인 업사이즈 추천  
  - 보정 문구: 앞코 너비 보정 필요 시 명시

- Q5-3 = `1(네)`  
  - 추천사이즈: 보수 사이즈(예: `225mm`)  
  - 설명: 업사이즈 시 헐떡임 발생 이력으로 보수 추천  
  - 보정 문구: 꽉낌 대응 보정 + 앞코 너비 보정 필요 명시

- Q5-3 = `3(신어본 적 없음·모름)`  
  - 설명: 불확실 케이스 분기  
  - 무지외반 통증 심함이면 업사이즈 우선 검토  
  - 작은 사이즈 선택 시 `발볼늘림 3단계 + 앞코 너비 보정` 안내

## 6) 보정 단일 적용 원칙 (Round 1 / 5-B 회귀 반영, 2026-05-10)

### 6.1 사용자 선호 핏라인 유지 원칙

- 입력된 선호 핏(Q5-0)을 시스템이 **임의로 상향하지 않는다**.
- 핏 상향은 다음 중 하나의 조건에서만 발생하며 그 사유를 `reason` 필드에 명시한다.
  - `_recommend_product`의 의학적 권고 분기 (예: 무지외반 심함, EE/EEE 폭 강제)
  - `_apply_experience_conservative_fit` (헐떡임 등 보수 보정, **꽉낌 제외**)
- 꽉낌(Q5-1=3) 분기에서는 핏라인을 변경하지 않는다.

### 6.1.1 핏라인 등가 — 「편한핏 이상」(2026-05-12 운영 정정)

- **편한핏**과 **아주 편한핏**은 추천·검증 축에서 **동일 계열**로 본다(둘 다 「편한핏 이상」). TC09 등 꽉낌 적용점수 2점 케이스에서 **편한핏·240**과 **아주 편한핏·240**은 동일 합격 해석으로 둔다.
- **발볼 늘림 1단계와 2단계는 서로 등가가 아니다.** 보정 강도·단계 수는 §6.2 환산표와 차감 규칙만 따르며, 발볼 단계 숫자를 바꿔 핏라인 등가로 설명하지 않는다.

### 6.2 보정 옵션 등가표

꽉낌·과보정 회피는 “총 보정 강도 N단계”를 **세 보정 단위의 합산이 N단계를 넘지 않도록** 분배한다.

보정 단위 환산:

| 보정 단위 | 등가 강도 |
|---|---|
| 사이즈 업 `+5mm` | **2단계** (신발 길이가 늘면서 발볼 둘레도 함께 늘어남) |
| 핏 한 단계 상향 (기본핏 → 편한핏 등) | 1단계 |
| 발볼 늘림 1단계 (2.5mm) | 1단계 |
| 핀포인트 가공 (무지외반 돌출부, 앞코 너비 늘림 권고 등) | 0단계 (별도 의학적 가공/권고로 점수에 포함하지 않음) |

총 보정 강도(N) 별 대표 처방:

| N | 대표 옵션 | 비고 |
|---|---|---|
| 1 | 사이즈 유지 + 발볼 늘림 1단계 (또는 핏 1단계 상향) | 옵션 C 단독 |
| 2 | 사이즈 +5mm (사이즈 업이 2단계 등가) **또는** 발볼 늘림 2단계 | 옵션 A 단독 또는 옵션 C 단독 |
| 3 | 사이즈 +5mm + 발볼 늘림 1단계 **또는** 발볼 늘림 3단계 | 옵션 A 단독으로는 부족, A+C 1단계 또는 C 3단계 |

규칙:
- 한 케이스 내에서 **모든 보정 단위(사이즈 업·핏 상향·발볼 늘림)의 합이 N단계를 넘지 않도록 차감**한다.
  - 예 1: 235→240 사이즈 업이 2단계분을 가져갔다면, 같은 케이스에서 발볼 늘림은 `N - 2`단계만 추가.
  - 예 2: 편한핏 사용자가 아주편한핏을 추천받았다면 핏 상향 1단계분이 이미 소진되었으므로 발볼 늘림은 `N - 1`단계만 추가.
  - 예 3 (TC04): 235·편한핏·무지외반 심함(N=3) → 사이즈 +5mm(2점) + 핏 한 단계 상향(1점) = 3점 → 발볼 늘림 0단계 + 핀포인트 무지외반 늘림 + 앞코 너비 늘림 권고.
- **“발볼 늘림 1단계는 사용자 선호 핏을 유지할 때만 추가”**: 핏을 한 단계 이상 상향한 케이스에 발볼 늘림까지 동시에 적용하면 등가 합산 오버.
- 핀포인트 가공은 점수에 포함하지 않으므로 무지외반 심함·앞코 가공 등은 **별도 트리거**로 부착한다. 무지외반 심함(hallux=3)은 `핀포인트 무지외반 늘림`과 `앞코 너비 늘림 권고`를 함께 안내한다.
- “사이즈 +5mm + 발볼 늘림 3단계” 같이 **사이즈 업과 발볼 늘림 강도가 동시에 N단계 이상**으로 적용되거나, **사이즈 업 + 핏 상향 + 발볼 늘림이 동시 적용**되면 오버사이즈 치명 오류로 분류한다.

### 6.3 Q5-3 응답별 처방 (단일 옵션 강제 + 등가 차감)

| Q5-3 응답 | heel_slip 값 | 적용 옵션 | 비고 |
|-----------|--------------|-----------|------|
| 2 (아니요) | False | **옵션 A** (사이즈 +5mm) | 사이즈 업이 2단계 등가. 발볼 늘림은 `max(0, N - 2)`단계까지만 추가 |
| 1 (네)     | True  | **옵션 C** 1단계 또는 2단계 | 사이즈 보수 유지, 핏 상향 금지 |
| 3 (신어본 적 없음·모름)   | None  | hallux=3: 옵션 A + 핀포인트 가공 + 앞코 너비 늘림 권고 / 그 외: 옵션 C n단계(적용점수) | 핀포인트 가공/권고는 점수와 무관하게 추가. 데모 칩: **「신어본 적 없음」**; 자유 입력 `잘 모르겠어요` 등은 동일 처리(`heel_slip=None`) |

### 6.4 3점 이상 케이스의 업사이즈 헐떡임 가드 (오버사이즈 방지)

- 적용 점수 N ≥ 3인 경우, 사이즈 업을 단독으로 처방하면 신발 길이가 늘어나 헐떡임 위험이 커진다.
- 다음 두 가지를 반드시 점검한다.
  1. **사이즈 업 시 헐떡임 응답을 받았는지 확인** (`heel_slip_when_one_size_up`이 `None`이면 안전측: 사이즈 업을 강제하지 않거나, 사이즈 업 + 발볼 늘림 1단계로 분할).
  2. **사이즈 업 + 발볼 늘림 N단계 동시 적용 금지.** 사이즈 업이 이미 2단계를 차지하므로 발볼 늘림은 `max(0, N - 2)`단계로 줄인다.
- 위 규칙을 어기면 오버사이즈 치명 오류로 분류한다 (`docs/portfolio/04_debug_cases.md` 참고).

### 6.4.1 Q5-3과 중복되는 추가 질문안 (철회, 2026-05-12)

- TC10 수동 검토 중 **「한 치수 더 크게 신었을 때 체감」**을 별도 질문으로 두는 안이 검토되었으나, 현행 **Q5-3** 질문(지금 사이즈보다 한 치수 크게 신었을 때 뒤꿈치 헐떡임 여부·**신어본 적 없음**)과 **질문 의도가 겹쳐** 사용자 입장에서 **중복 이슈**가 발생한다.
- 따라서 해당 추가 질문은 **도입하지 않는다.** 업사이즈·헐떡임 가드는 **§6.4 + Q5-3(`heel_slip_when_one_size_up`)**만으로 유지한다.
- 관련 백로그 항목(`09_ideas_backlog.md` 2026-05-12)은 **철회(중복)** 처리한다.

### 6.5 위반 사례 (Round 1 디버그)

- **TC03·TC09·TC10**: heel_slip=False 시 사이즈 +5mm 정상이지만 동시에 핏라인까지 한 단계 위로 상향됨 → 옵션 A에 옵션 B가 합산
- **TC04 (1차 핫픽스 후 2차 발견)**: 무지외반 심함(3점) 처방에서 사이즈 +5mm(2점) + 핏 한 단계 상향(1점) = 3점으로 보정 강도가 이미 충족되는데도 발볼 늘림 1단계를 추가 적용 → 등가 합산 4단계로 1점 오버사이즈 위험. 정답은 `사이즈 240 + 아주편한핏 + 발볼 늘림 0단계 + 핀포인트 무지외반 늘림 + 앞코 너비 늘림 권고`. (발볼 늘림 1단계를 처방하려면 핏을 사용자 선호 `편한핏`으로 유지하는 분기여야 함.)
- **TC08**: heel_slip=None + 적용점수 1 시 핏 상향(B) + 발볼 늘림 1단계(C) 동시 적용 → B와 C 합산
- 회복 기준: 위 케이스에서 진단 결과의 `recommended_fit`이 사용자 선호 핏(또는 `_recommend_product`가 산출한 값)과 동일하고, `final_size`와 `stretch_step`의 합산 강도가 위 6.2/6.3/6.4 표를 넘지 않음

### 6.6 다중 옵션 안내 (선택)

진단이 단일 옵션으로 확정된 경우라도, 사용자에게 **등가 후보 2~3개를 함께 안내**할 수 있다 (`09_ideas_backlog.md` 2026-05-09 항목). 단, 시스템 진단(`final_size`, `recommended_fit`, `stretch_step`)은 **반드시 하나로 확정**한다.
