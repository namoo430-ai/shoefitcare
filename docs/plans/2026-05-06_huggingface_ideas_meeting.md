# Hugging Face · ML 확장 방향 정리

**일자:** 2026-05-06  
**성격:** 아이디어 회의 및 적용 가능성·리스크·권장 방식 합의 초안.

---

## 1) 발 이미지 기반 진단 (CV)

**적용 가능 여부:** 가능 (초기에는 **보조 신호**로 권장)

**기술 메모**

- SegFormer / Mask2Former로 윤곽 추출 → 비율 기반 특징값 산출
- 카드 기준점(DETR) 등으로 실측 추정 검토 가능

**주요 리스크**

- 촬영 품질(각도·거리·조명)에 따라 오차 큼
- 기준 물체 미포함 시 절대치 추정 불안정
- 생체·민감정보 취급: 동의·보관 정책 필수

**권장 적용 방식**

- “확정 진단” 금지, **보조 점수**로만 사용
- 사진 품질 검사 + 재촬영 가이드 선행
- 저장 최소화: 특징값 위주, 원본은 단기 보관 또는 즉시 폐기 옵션

---

## 2) 대화형 고민 상담 (NLP)

**적용 가능 여부:** 매우 높음 (단기 효과 큼)

**기술 메모**

- KoBERT / RoBERTa 기반 **의도 분류** + **증상 엔티티 추출**
- 현재 `session` 파서 보완에 직접 효과

**주요 리스크**

- 도메인 데이터 부족 시 분류 오탐
- 신조어·구어체·오타 대응 필요

**권장 적용 방식**

- 룰 기반 파서 + NLP 분류 **하이브리드**
- confidence 낮으면 기존 흐름(재질문)으로 **fallback**
- 주간 라벨링으로 지속 개선

---

## 3) 개인화 추천 (Recommender)

**적용 가능 여부:** 높음 (중기 핵심)

**기술 메모**

- 임베딩 유사도로 “비슷한 고민 사용자 만족 모델” **가점**
- 제로샷으로 초기 태깅·콜드스타트 완화 가능

**주요 리스크**

- 리뷰 편향(극단 리뷰, 브랜드 편향)
- 설명가능성 저하(왜 추천됐는지 불투명)

**권장 적용 방식**

- 기존 결정 엔진은 유지, 임베딩·제로샷은 **보조 가점**
- 추천 근거 출력 필수(“어떤 특징이 일치했는지”)
- 리뷰 신뢰도 필터(길이·중복·평점 극단치 보정)

---

## 4) Spaces / API 배포

**적용 가능 여부:** 매우 높음 (프로토타입에 적합)

**기술 메모**

- Spaces(Gradio / Streamlit)로 실험 UI 빠른 검증
- Inference API로 서버 부담 감소

**주요 리스크**

- 비용·지연·쿼터 한계
- 외부 의존 증가(서비스 정책 변경 리스크)

**권장 적용 방식**

- 프로토타입은 HF, **운영은 자체 API 우선**
- 핵심 경로는 항상 **fallback**(기존 엔진 경로)

---

## 우리 엔진 기준 최적 아키텍처 (합의)

| 역할 | 담당 |
|------|------|
| **결정** | 현재 **규칙·점수 엔진** (재현성 유지) |
| **보강** | HF NLP/CV는 **feature provider**로 입력 신호 추가 |
| **설명** | RAG/LLM으로 **근거 설명** 강화 |

즉 **“엔진 교체”가 아니라 엔진 강화** 전략이 맞음.

## 확장성 판정

| 구분 | 판단 |
|------|------|

| 운영 리스크 | 중간 — CV 품질·개인정보·외부 API 의존 관리 필요 |
