#### 방향성 점검표 (3개 핵심 지표)

- 목적: "진단 엔진이 잘 가는 중인지"를 라운드 단위로 숫자로 확인
- 기록 시점: Round 시작 시 Baseline 1회, Round 종료 시 Current 1회

| 지표 | 정의 | Baseline | Current | 목표/판정 |
|---|---|---|---|---|
| 변경 비용 | 새 정책/규칙 1건 반영 시 수정 파일 수 + 수정 LOC |  |  | 감소 추세면 양호 |
| 품질 지표 | 동일 입력 재실행 시 일관성(`all_ok=true`) + 회귀 실패율(`실패 TC/전체 TC`) |  |  | 일관성 상승 + 실패율 하락 |
| 운영 지표 | 평균 장애 대응 시간(MTTR, 분) + 수동 보정 횟수(주간) |  |  | MTTR/보정 횟수 하락 |

- 간단 계산식:
  - 변경 비용 점수: `수정 파일 수 + (수정 LOC / 50)` (낮을수록 좋음)
  - 품질 실패율: `(Fail + Partial) / 전체 TC * 100`
  - 운영 개선율: `(Baseline - Current) / Baseline * 100`

- 해석 가이드(간단):
  - 3개 중 2개 이상 개선: 방향성 적합
  - 3개 중 2개 이상 악화: 정책/구현/데이터 중 원인 분리 후 우선 재조정
