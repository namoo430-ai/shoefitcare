# 파일럿 룰 엔진 (Q1~Q4 → SF00~SF05)

버전: 20260619 (`pilot_engine.PILOT_RULE_VERSION`)

## Q1 (단일 선택, 4지)

1. 길이는 맞는데 발볼이 꽉 끼어요
2. 발볼 때문에 한사이즈 크게 주문하면 길이가 헐떡거려요
3. 발등이 눌려서 불편해요
4. **평소 대부분 신발이 헐거운 편이에요** (구 Q5 「헐거운 편」 통합)

「특별한 불편 없음」은 Q1이 아니라 **Q2·Q3**에서 처리한다.

## Q4 사이즈

225, 230, 235, 240, 245, 250, 255 (5mm)

## Q2 규칙

「특별한 불편 없음」은 **단독만** 선택 가능 (다른 항목과 동시 불가).

## 판정 우선순위

1. Q3 == 「특별한 불편 없음」 → **SF00**
2. Q1 == 「길이…발볼 조임」 AND Q2 **발볼만** AND Q3 == 「매우 불편」 → **SF02**
2b. Q1 == 「길이…발볼 조임」 AND Q3 == 「매우 불편」(2번 해당 없음) → **SF03**
3. Q1 == 「발등…」 AND Q2에 「무지외반」 AND Q3 == 「매우 불편함」 → **SF03**
4. Q1 == 「발등…」 AND Q2 **발등만** AND Q3 == 「매우 불편함」 → **SF03** (한 사이즈 업)
4b. Q1 == 「발등…」 AND Q2에 「발등」+다른 부위 AND Q3 == 「매우 불편함」 → **SF03** (복합·심함; **정밀 완료 시 SF05**)
4. Q2에 「무지외반」 AND Q3 == 「매우 불편함」 (2·3번에 해당 안 됨) → **SF02**
5. Q1 == 「헐떡」 AND Q2에 「발볼」 AND Q3 == 「매우 불편함」 → **SF02** (헐떡 일반 SF01보다 우선)
6. Q1 == 「길이는 맞는데 발볼이 꽉 끼어요」 AND Q3 in (약간, 보통) → **SF01**
7. Q1 == 「발등…」 AND Q2에 「발등」(단독·중복) AND Q3 in (약간, 보통) → **SF01** (발등 맞춤 안내)
7b. Q1 == 「발등…」 AND Q2에 「발등」 **없음** AND Q3 in (약간, 보통) → **SF01** (일반 1단계)
7c. Q1 == 「발등…」 AND Q2에 「발등」 **없음** AND Q3 == 매우 → **SF03** (4·5번보다 뒤: Q2 무지외반+매우는 **SF02** 유지)
8. Q1 == 「발볼 때문에 한사이즈 크게…헐떡」 (위 5번 해당 없음) → **SF01**
9. Q1 == 「평소 대부분 신발이 헐거운 편」 → **SF04-A** (R3 좁은발 참고; 간편 결과는 **칼발·좁은발(길이맞음)** 두 경우 tip 병기)  
   - **칼발에 가까움**: 길이 때문에 한 사이즈 크게 → 헐거움 → 기본핏 + 앞깔창  
   - **좁은발(길이 맞음)**: 정사이즈인데도 여유/헐거움 → 기본핏 + 앞깔창  
   - 정밀 완료 시 **SF04-A만** `SF05` 승격 (칼발·길이/앞깔창 문구). 간편 화면에 「칼발」 코드는 두지 않음.
10. Q1 == 「발등…」 + Q2 발등+다른 부위 + Q3 매우 → **SF04-B** (발등 복합·심함) — **4문 진단 종료 코드는 SF04 유지** (SF05 아님). 정밀 탭은 복합·S3 등으로 **선택** 가능하나, 정밀 제출로 SF05 승격하지 **않음** (2026-06-19).
11. 그 외 → **SF00**

구 5문항: Q5 헐거움 + Q1 꽉 → SF01 규칙은 **폐지** (Q1 단일 선택으로 대체).

## SF04 vs SF05 (팀 합의 · 단순 모델)

| 코드 | 의미 |
|------|------|
| **SF04** | 4문 **칼발·좁은발(헐거움)** — Q1 「대부분 여유」(+ 레거시 q5 헐거움) |
| **SF05** | **정밀 진단 완료** (발 치수·사진 접수 후). 4문만으로는 SF05 없음 |

발등 복합·심함 등은 4문에서 **SF03** 등으로 끝나고, 정밀이 필요하면 탭에서 들어가 **완료 시 SF05**.

| 구분 | 4문 결과 | 정밀 완료 후 |
|------|----------|--------------|
| 헐거움·좁은발 | **SF04** | **SF05** |
| 발등 복합·심함 | **SF03** | **SF05** |
| SF01~03 등 | 해당 SF | **SF05** |

**SF05 ≠ 「복합 증상 자동 코드」** — 정밀을 **마친** 고객의 관리·문구용 코드.

## 안내 문구 (발볼 늘림)

- **Q1 헐떡** 경로에서 SF01·SF02: Q4(평소 사이즈) 기준 **한 사이즈 작게(5mm)** + 해당 **발볼 늘림 단계**를 안내 (제목에 「헐떡임 대응」).
- **Q1 발등** + Q2 발등 + Q3 약간/보통 → SF01 안내: 「전체적인 압박을 완화하기 위해 발볼 늘림 1단계」
- **Q1 발등** + Q2 **발등 없음** + Q3 약간/보통 → **SF01** (일반 1단계 문구)
- **Q1 발등** + Q2 발등 **만** + Q3 매우 → **SF03** (한 사이즈 업)
- **Q1 발등** + Q2 발등 + 다른 부위 + Q3 매우 → **SF04**

## 정밀

- 고객 UI `precision_recommended`는 **항상 false** (간편진단·문의 복사 우선). 정밀 폼은 결과 화면 **선택** 링크로만 진입.

## 반품 코호트 (관리자)

| 그룹 | 정의 |
|------|------|
| 진단없음 | `pilot_orders` 중 `has_diagnosis=0` |
| 진단만 | 진단 있음, `recommendation_code` not in (SF01,SF02) and `actual_work_step` not in (1,2) |
| 진단+발볼늘림 | SF01/SF02 또는 `actual_work_step` 1·2단계 |

Lite 2문 진단 경로는 파일럿 검증에서 **제외** (`/pilot` 4문만 사용).
