# 파일럿 룰 엔진 (Q1~Q4 → SF00~SF05)

버전: 20260605

## Q1 (단일 선택, 4지)

1. 길이는 맞는데 발볼이 꽉 끼어요
2. 발볼 때문에 한사이즈 크게 주문하면 길이가 헐떡거려요
3. 발등이 눌려서 불편해요
4. **평소 대부분 신발이 헐거운 편이에요** (구 Q5 「헐거운 편」 통합)

「특별한 불편 없음」은 Q1이 아니라 **Q2·Q3**에서 처리한다.

## Q4 사이즈

225, 230, 235, 240, 245, 250, 255 (5mm)

## Q2 규칙

「특별한 불편 없음」은 **단독만** 선택 가능 (다른 항목과 동시 불가).

## 판정 우선순위

1. Q3 == 「특별한 불편 없음」 → **SF00**
2. Q1 == 「길이는 맞는데 발볼이 꽉 끼어요」 AND Q3 == 「매우 불편함」 → **SF03** (무지외반 **있음/없음** 모두 SF03)
3. Q1 == 「발등…」 AND Q2에 「무지외반」 AND Q3 == 「매우 불편함」 → **SF03**
4. Q1 == 「발등…」 AND Q2에 「발등」(무지외반 **없음**) AND Q3 == 「매우 불편함」 → **SF04** (Q2 복합이면 정밀 권장)
4. Q2에 「무지외반」 AND Q3 == 「매우 불편함」 (2·3번에 해당 안 됨) → **SF02**
5. Q1 == 「헐떡」 AND Q2에 「발볼」 AND Q3 == 「매우 불편함」 → **SF02** (헐떡 일반 SF01보다 우선)
6. Q1 == 「길이는 맞는데 발볼이 꽉 끼어요」 AND Q3 in (약간, 보통) → **SF01**
7. Q1 == 「발등…」 AND Q2에 「발등」(단독·중복) AND Q3 in (약간, 보통) → **SF01** (발등 맞춤 안내)
7b. Q1 == 「발등…」 AND Q2에 「발등」 **없음** AND Q3 in (약간, 보통) → **SF01** (일반 1단계)
7c. Q1 == 「발등…」 AND Q2에 「발등」 **없음** AND Q3 == 매우 → **SF03** (4·5번보다 뒤: Q2 무지외반+매우는 **SF02** 유지)
8. Q1 == 「발볼 때문에 한사이즈 크게…헐떡」 (위 5번 해당 없음) → **SF01**
9. Q1 == 「평소 대부분 신발이 헐거운 편」 → **SF04** (좁은 발 유형; 레거시 `q5` 헐거움 동일)
10. 그 외 → **SF00**

구 5문항: Q5 헐거움 + Q1 꽉 → SF01 규칙은 **폐지** (Q1 단일 선택으로 대체).

## 안내 문구 (발볼 늘림)

- **Q1 헐떡** 경로에서 SF01·SF02: Q4(평소 사이즈) 기준 **한 사이즈 작게(5mm)** + 해당 **발볼 늘림 단계**를 안내 (제목에 「헐떡임 대응」).
- **Q1 발등** + Q2 발등 + Q3 약간/보통 → SF01 안내: 「전체적인 압박을 완화하기 위해 발볼 늘림 1단계」
- **Q1 발등** + Q2 **발등 없음** + Q3 약간/보통 → **SF01** (일반 1단계 문구)
- **Q1 발등** + Q2 발등 + Q3 매우 → **SF04** (발등 Q2 일치 시; 복합 Q2면 정밀 권장)
- **Q1 발등** + Q2 **발등 없음** + Q3 매우 → **SF03** (한 사이즈 업·정밀 CTA)

## 정밀

- SF04 + 정밀 완료 → **SF05** 로 갱신
- 정밀: 좌/우 발길이·발볼 (입력 cm, 저장 mm), 연락처 + 동의 문구

## 반품 코호트 (관리자)

| 그룹 | 정의 |
|------|------|
| 진단없음 | `pilot_orders` 중 `has_diagnosis=0` |
| 진단만 | 진단 있음, `recommendation_code` not in (SF01,SF02) and `actual_work_step` not in (1,2) |
| 진단+발볼늘림 | SF01/SF02 또는 `actual_work_step` 1·2단계 |

Lite 2문 진단 경로는 파일럿 검증에서 **제외** (`/pilot` 4문만 사용).
