상태: 채택  

제목: 추천 코어는 규칙 엔진 유지, HF·ML은 feature provider·설명 보강

결정: **현재 규칙·점수 엔진**이 추천·재현성의 코어를 담당한다. HF NLP/CV는 **입력 feature·보조 점수**로만 연결하고, **설명·근거**는 RAG/LLM 경로로 강화한다. CV 발 이미지는 **확정 진단 금지·보조만**, NLP는 **룰+NLP 하이브리드·저신뢰 시 fallback**.  

대안: HF 모델로 엔진 단일화·랭킹 전면 교체  

근거: 재현성·운영 통제·데이터 규모·민감정보 리스크·외부 API 의존  

결과/재검토일: 2026-09-01 (CV/NLP 가세 방식·비용·지표 반영 후)

ID: ADR-2026-05-002

날짜: 2026-05-07

상태: 채택

제목: 톡톡 양방향 챗봇 연동 보류, 비공개 웹 데모로 엔진 정교화 선행

배경: 톡톡 알림 API/챗봇 API 혼재로 간헐 미유입·빈 payload 발생, 운영 디버깅 비용 큼. 추천 엔진 출력 품질도 아직 일관되지 않아 고객 노출에는 부적합.

결정: 1) 네이버 톡톡 **양방향 챗봇 연동은 현재 보류**한다. 2) 운영/시연/내부 검증 채널은 **웹 데모 `/demo`로 한정**한다 (외부 공개 X). 3) `docs/plans/TESTSET_20_TEMPLATE.md` 기반으로 **룰·데이터·설명 위주 정교화**를 우선 진행하고, 자유입력 보강이 필요해지는 시점에 NLP 보조를 도입한다. 4) **엔진 정교화 기준 충족 후** 톡톡 챗봇 API 권한 신청을 진행한다.

대안: (a) 톡톡 양방향을 계속 시도하며 동시에 엔진 개선  (b) 알림 API만 사용하고 톡톡에서는 안내문만 노출

근거: 권한 의존성으로 단기간 양방향 안정화 불가 / 엔진 출력이 불안정한 상태에서의 고객 노출은 신뢰 저하 위험 / 채널 변경과 엔진 변경을 분리해야 회귀 분석 가능 / 시간 한계

결과/재검토일: **2026-05-21 재검토** 또는 testset 정확도 80% 도달 시 즉시 재검토
