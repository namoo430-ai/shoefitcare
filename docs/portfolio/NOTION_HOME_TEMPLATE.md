## 데이터베이스 ① 회의록

### 가이드 표 vs 실제 DB (중요)

- **아래 마크다운 표**는 “DB에 어떤 **열(속성)**을 만들지” 설명용입니다. 이 표 자체를 매번 복불해 고치면 안 됩니다.
- **실제 회의 기록**은 노션 **데이터베이스 블록**을 하나 만든 뒤, 그 표에서 **`+ 새로 만들기`로 행만** 추가합니다.

**보기 추천:** 테이블 → 보드(상태별) 또는 캘린더(날짜별)

#### (참고) 속성 설계 — 열 이름·타입

| 속성 이름   | 타입     | 설명 |
|------------|----------|------|
| 이름       | 제목     | 회의 제목 |
| 날짜       | 날짜     | 회의일 |
| 참석       | 사람/텍스트 | 선택 |
| 안건       | 텍스트   | 다줄 |
| 결정 사항  | 텍스트   | 다줄 |
| 다음 할 일 | 텍스트   | 체크리스트로 써도 됨 |
| 링크       | URL      | GitHub 이슈, PR, 커밋, 노션 하위 페이지 |
| 상태       | 선택     | 예: 기록완료 / 후속필요 |

**입력 규칙:** 회의 끝나면 10분 안에 1행 추가. 장문 기술 증빙은 Git `03_dev_log`에 맡기고, 여기엔 결정·To-do만.

#### 실제 회의록 DB 만들기 (순서)

1. 위 **참고 표 바로 아래**에 빈 줄을 하나 둡니다.
2. `/데이터베이스` 입력 → **인라인** 또는 **전체 페이지** 중 하나 선택  
   - 홈 한 페이지에 모아두려면 **인라인**이 보통 편합니다.
3. 생긴 DB 표 오른쪽 **`+`** 또는 **`속성`**으로 열을 추가합니다.  
   - 위 표와 같이: `날짜`, `참석`, `안건`, `결정 사항`, `다음 할 일`, `링크`, `상태` 등  
   - (노션 기본 **이름** 열이 곧 “회의 제목” 역할입니다.)
4. **첫 행**에 예시 회의 하나를 적어 두면 이후에 형식이 안 헷갈립니다.
5. 다음 회의부터는 표 맨 아래 **`+ 새로 만들기`**만 눌러 **한 행씩** 추가합니다.  
   - 위 “참고 표”는 **손대지 않고** 그대로 두면 됩니다.

|----------|--------|------|
| 이름     | 제목   | 짧은 결정 한 줄 |
| 날짜     | 날짜   | |
| 배경     | 텍스트 | 왜 결정이 필요했는지 |
| 결정     | 텍스트 | 무엇을 했는지 |
| 대안     | 텍스트 | 고려했던 다른 선택 |
